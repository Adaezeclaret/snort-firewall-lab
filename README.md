# Snort and Firewall Rules Lab

## Project Overview

This project documents the completion of the Cisco Networking Academy **Snort and Firewall Rules** lab. The objective was to gain hands-on experience with network intrusion detection and firewall configuration in a virtualized environment.

The lab involved configuring a simulated enterprise network using Mininet, monitoring network traffic with Snort IDS, capturing malicious network activity with tcpdump, and implementing firewall rules using iptables to block malicious traffic.

This repository contains screenshots, packet captures, commands executed during the lab, and documentation of the activities performed.

## Objectives

- Configure the Cisco Security Workstation virtual environment.
- Launch and manage the Mininet network topology.
- Monitor network traffic using Snort Intrusion Detection System (IDS).
- Capture network traffic using tcpdump.
- Download and inspect simulated malicious traffic.
- Observe and analyze Snort alerts.
- Configure iptables firewall rules to block malicious traffic.
- Verify that firewall rules successfully prevented access to the malicious server.

## Lessons Learned

Completing this lab helped me understand how signature-based intrusion detection works in practice. Observing Snort generate alerts for malicious traffic demonstrated how IDS solutions inspect packet payloads rather than simply relying on packet headers.

I also gained practical experience using tcpdump to capture network traffic and configuring firewall rules with iptables to block malicious connections. This reinforced the relationship between detection (IDS) and prevention (firewall) in a layered network defense strategy.
